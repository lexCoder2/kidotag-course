## El principio 12-Factor

dotenv implementa el [factor III de la metodología 12-Factor](https://12factor.net/config): **"Store config in the environment"**.

La regla es simple: **si el valor cambiaría entre tu laptop, el servidor de staging y producción, es configuración — no código**.

| ¿Es configuración?  | Ejemplos                                               |
| ------------------- | ------------------------------------------------------ |
| ✅ Sí, usa .env     | URL de BD, JWT secret, puerto, claves de API           |
| ❌ No, en el código | Rutas de Express, lógica de negocio, componentes React |

## El archivo `.env` de kidotag10

```bash
# api/.env — basado en ESCUELA_DEMO.md del proyecto
PORT=4000
MONGODB_URI=mongodb://localhost:27017/kidotag_demo
JWT_SECRET=kidotag_secret_cambiar_en_produccion
FRONTEND_URL=http://localhost:5173

# Datos de la escuela demo
ESCUELA_NOMBRE="Escuela Demo Kidotag"
ESCUELA_ADMIN_EMAIL=admin@kidotag.demo
ESCUELA_ADMIN_PASS=Admin2024!
```

<Callout variante="danger" titulo=".env va en .gitignore">
  El archivo `.env` contiene credenciales reales. **Nunca** debe subirse a git.
  Verifica que esté en tu `.gitignore` antes de hacer commit. kidotag10 incluye
  un archivo `.env.example` como plantilla.
</Callout>

## Cómo acceder a las variables

```javascript
// En cualquier archivo del API (después del require('dotenv').config())
const { MONGODB_URI, JWT_SECRET, PORT } = process.env;

// Con valores por defecto para desarrollo
const port = process.env.PORT || 4000;
const secret = process.env.JWT_SECRET;

if (!secret) {
  console.error("❌ JWT_SECRET no definido en .env");
  process.exit(1); // no arrancar sin el secret
}
```

<Callout variante="tip" titulo="process.env siempre son strings">
  `process.env.PORT` es siempre un string, aunque escribas `PORT=4000` en el
  .env. Si necesitas un número, conviértelo: `Number(process.env.PORT) || 4000`.
</Callout>

## src/config/database.js de kidotag10

```javascript
const mongoose = require("mongoose");

async function conectarDB() {
  try {
    await mongoose.connect(process.env.MONGODB_URI);
    console.log("✅ MongoDB conectado");
  } catch (error) {
    console.error("❌ Error de conexión:", error.message);
    process.exit(1);
  }
}

module.exports = { conectarDB };
```
