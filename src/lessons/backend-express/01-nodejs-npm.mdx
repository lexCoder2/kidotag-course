## Node.js y el modelo de ejecución

Node.js usa un **modelo de un solo hilo con event loop**. Esto significa que:

- Las operaciones de I/O (base de datos, archivos, red) son **asíncronas**
- El servidor puede manejar miles de conexiones simultáneas sin bloquear
- Por eso todo en kidotag10 usa `async/await` para operaciones de DB

```javascript
// ✅ Así opera kidotag10 — async/await
async function obtenerAlumnos(req, res) {
  const alumnos = await Alumno.find(); // no bloquea el hilo
  res.json({ ok: true, data: alumnos });
}

// ❌ Bloquea el event loop — nunca usar en producción
const datos = fs.readFileSync("archivo-enorme.json"); // bloquea todo
```

## El archivo `package.json`

Todo proyecto Node.js tiene un `package.json`. Es el "currículum" del proyecto:

```json
// api/package.json de kidotag10
{
  "name": "api",
  "version": "1.0.0",
  "main": "src/index.js",
  "scripts": {
    "dev": "nodemon src/index.js",
    "start": "node src/index.js"
  },
  "dependencies": {
    "express": "^5.2.1",
    "mongoose": "^9.1.4",
    "jsonwebtoken": "^9.0.2",
    "bcryptjs": "^2.4.3",
    "cors": "^2.8.5",
    "socket.io": "^4.7.2",
    "dotenv": "^17.2.3",
    "swagger-jsdoc": "^6.2.8",
    "xlsx": "^0.18.5"
  },
  "devDependencies": {
    "nodemon": "^3.1.11"
  }
}
```

<Callout variante="info" titulo="dependencies vs devDependencies">
  **dependencies** se instalan en producción y en desarrollo.
  **devDependencies** solo en desarrollo. `nodemon` solo se necesita para
  programar, no para ejecutar la app en el servidor real.
</Callout>

## Módulos CommonJS

kidotag10 API usa el sistema de módulos **CommonJS** (el original de Node.js):

```javascript
// Importar (require)
const express = require("express");
const { Router } = require("express");

// Exportar (module.exports)
module.exports = router;
module.exports = { conectarDB, cerrarDB };
```

<Callout variante="tip" titulo="CommonJS vs ES Modules">
  El frontend de kidotag10 (Vite + React) usa **ES Modules**
  (`import`/`export`). El backend usa **CommonJS** (`require`/`module.exports`).
  Son dos sistemas distintos — en un mismo proyecto puedes tener ambos según el
  entorno.
</Callout>
