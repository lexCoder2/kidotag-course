**JWT** (JSON Web Token) es el sistema de autenticación sin estado que usa kidotag10. En lugar de guardar sesiones en el servidor, toda la información necesaria va **dentro del token**.

## Estructura del token

Un JWT tiene 3 partes separadas por puntos:

```
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOiI2NGFiMTIiLCJyb2wiOiJwcm9mZXNvciIsImV4cCI6MTcxNTAwMDAwMH0.xK9p2mN7
   ──────────────────  ─────────────────────────────────────────────────────────────────────────────  ────────
        HEADER                              PAYLOAD                                      SIGNATURE
```

- **Header**: algoritmo de firma (`HS256`)
- **Payload**: datos del usuario (claims)
- **Signature**: HMAC-SHA256 del header + payload con la clave secreta

## Claims en kidotag10

```javascript
// auth.controller.js — jwt.sign()
const token = jwt.sign(
  {
    userId: usuario._id, // Para buscar el usuario en cada request
    rol: usuario.rol, // 'profesor' | 'tutor' | 'admin'
    escuelaId: usuario.escuelaId,
    // NO incluir: password, datos sensibles
  },
  process.env.JWT_SECRET,
  { expiresIn: "7d" },
);
```

<Callout variante="warning" titulo="JWTs son firmados, no encriptados">
  El payload de un JWT puede ser leído por cualquiera (es solo Base64URL). La
  **firma** garantiza que no fue modificado, pero no esconde los datos. Nunca
  incluyas passwords, datos médicos o información sensible en el payload.
</Callout>
